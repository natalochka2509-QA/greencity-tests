# QA Homework - GreenCity Events Testing

## 📋 Опис завдання

Це завдання передбачає розроблення набору тест-кейсів для функціональної перевірки сторінки подій веб-додатку **GreenCity**. 

Завдання включає:
- ✅ 3 позитивних тест-кейси для основного функціоналу
- ✅ 3 негативних тест-кейси для перевірки помилкових сценаріїв
- ✅ Дотримання єдиного формату оформлення
- ✅ Чіткий опис передумов, кроків та очікуваних результатів

## 🔗 Посилання на сторінку, що тестується

**GreenCity Events:** https://www.greencity.cx.ua/#/greenCity/events

## 📁 Структура файлів

- **test_cases.md** — основні тест-кейси (3 позитивних)
- **Dodatkove.md** — додаткові тест-кейси (3 негативних)
- **README.md** — документація проекту

## 🧪 Тестові сценарії

### Позитивні тест-кейси (test_cases.md)
1. Filter Events by Type — фільтрування подій за типом
2. Join an Event — приєднання користувача до відкритої події
3. Search and View Event Details — перегляд деталей про подію

### Негативні тест-кейси (Dodatkove.md)
1. Attempt to Join Event Without Authentication — тест без авторизації
2. Search Events with Empty or Invalid Search Query — тест з невалідними даними
3. Rate Event Without Login Credentials — тест оцінювання без логіну

## 📝 Формат тест-кейсів

Кожен тест-кейс містить:
- **Title** — назва тесту
- **Preconditions** — передумови виконання
- **Test Steps** — таблиця з колонками:
  - Step (номер кроку)
  - Action (дія)
  - Data (тестові дані)
  - Expected Result (очікуваний результат)
